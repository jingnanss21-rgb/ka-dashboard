🔄 KA 主看板反馈迁移工具

将 Cloudflare KV 中的异动反馈数据迁移到 Supabase，迁移过程完全可视化，不会丢失任何数据。

1 在 Supabase SQL Editor 执行建表语句

打开 Supabase SQL Editor， 粘贴以下 SQL 并执行（如果表已存在会跳过，不影响已有数据）：

-- 创建 KA 主看板异动反馈表 CREATE TABLE IF NOT EXISTS ka_alert_feedbacks ( id UUID DEFAULT gen_random_uuid() PRIMARY KEY, brand_name TEXT NOT NULL, cmp_type TEXT NOT NULL, -- 'mom' 或 'yoy' text TEXT NOT NULL, owner TEXT NOT NULL DEFAULT '未知', created_at TIMESTAMPTZ DEFAULT NOW(), updated_at TIMESTAMPTZ DEFAULT NOW(), UNIQUE(brand_name, cmp_type) -- 每个品牌每种类型只保留最新一条 ); -- 启用实时订阅 ALTER PUBLICATION supabase_realtime ADD TABLE ka_alert_feedbacks; -- 行级安全策略 ALTER TABLE ka_alert_feedbacks ENABLE ROW LEVEL SECURITY; CREATE POLICY "allow all ka_alert_feedbacks" ON ka_alert_feedbacks FOR ALL USING (true) WITH CHECK (true);

2 读取 KV 数据 & 预览

点击按钮从 Cloudflare KV 拉取所有反馈数据，先预览确认再迁移。

3 执行迁移

确认数据无误后，点击按钮写入 Supabase。已存在的品牌+类型组合会更新文字，不会重复插入。
